Deploy the Next.js frontend to Vercel for production hosting.

## Deploy

### From CLI

```bash
npx vercel --prod
```

### From dashboard

1. Import the GitHub repository at [vercel.com/new](https://vercel.com/new)
2. Framework preset: **Next.js**
3. Build command: `npm run build` (or leave the detected default)
4. Set environment variables (see below)

## Environment variables

Set these in **Vercel → Project → Settings → Environment Variables**:

| Variable | Required | Description |
|----------|----------|-------------|
| `MONGODB_URI` | Yes | MongoDB Atlas connection string; store as a secret |
| `MONGODB_DB` | No | Database name; defaults to `memory404` |
| `NEXT_PUBLIC_API_URL` | Yes | Render API URL |
| `NEXT_PUBLIC_SUPABASE_URL` | No | Supabase Auth project URL |
| `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_DEFAULT_KEY` | No | Supabase Auth publishable key |
| `CLOUDINARY_CLOUD_NAME` | No | Image upload |
| `CLOUDINARY_API_KEY` | No | Cloudinary API key |
| `CLOUDINARY_API_SECRET` | No | Cloudinary signing secret |

### Production values

```bash
NEXT_PUBLIC_API_URL=https://memory404-api.onrender.com
MONGODB_DB=memory404
NEXT_PUBLIC_SUPABASE_URL=https://<project-ref>.supabase.co
```

Add `MONGODB_URI` through Vercel's environment-variable UI; do not paste a real URI into source files or documentation. Use the same MongoDB database on Vercel and Render.

Supabase remains Auth-only. Next.js Route Handlers persist application records directly to the normalized MongoDB `users`, `groups`, and `links` collections.

## What Vercel serves

| Route | Handler |
|-------|---------|
| `/` | Main link vault |
| `/settings` | Settings page |
| `/trash` | Trash management |
| `/workspace` | Screen editor |
| `/api/*` | Next.js Route Handlers (trash, stats, image-proxy, health) |

## Redeploy

```bash
npx vercel --prod
```

Or push to the connected Git branch for automatic deployment.

## Custom domain

Add a custom domain in **Vercel → Project → Settings → Domains**. Update `CORS_ORIGINS` on Render to include the new domain.
