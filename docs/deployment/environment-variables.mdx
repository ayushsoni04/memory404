## Required

### Database

| Variable | Where | Description |
|----------|-------|-------------|
| `MONGODB_URI` | Next.js + Express | MongoDB Atlas connection string; store as a secret |
| `MONGODB_DB` | Next.js + Express | Database name; optional, defaults to `memory404` |

```bash
MONGODB_URI="mongodb+srv://<user>:<password>@<cluster>/<database>?retryWrites=true&w=majority"
MONGODB_DB="memory404"
```

Use a dedicated Atlas database user with only the permissions the app needs. Never prefix `MONGODB_URI` with `NEXT_PUBLIC_` or commit its value.

## Frontend

| Variable | Where | Description |
|----------|-------|-------------|
| `NEXT_PUBLIC_API_URL` | Vercel | Render API base URL. Empty = same-origin `/api` |
| `NEXT_PUBLIC_SUPABASE_URL` | Vercel | Supabase Auth project URL |
| `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_DEFAULT_KEY` | Vercel | Supabase Auth publishable key |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Vercel | Legacy Auth fallback anon key |

### Production example

```bash
NEXT_PUBLIC_API_URL=https://memory404-api.onrender.com
NEXT_PUBLIC_SUPABASE_URL=https://<project-ref>.supabase.co
```

Supabase is Auth-only. Application records are persisted in MongoDB Atlas.

## Backend (Express)

| Variable | Where | Description |
|----------|-------|-------------|
| `PORT` | Render | Server port (default `4000`) |
| `NODE_ENV` | Render | `production` |
| `CORS_ORIGINS` | Render | Comma-separated extra allowed origins |

```bash
CORS_ORIGINS=https://memory404.vercel.app
```

The Express service also requires `MONGODB_URI` and uses the same `MONGODB_DB` as Vercel.

## Optional

| Variable | Where | Description |
|----------|-------|-------------|
| `SCREENSHOT_URL_TEMPLATE` | Next.js + Express | Override screenshot provider URL |
| `CLOUDINARY_CLOUD_NAME` | Next.js + Render | Cloudinary cloud name |
| `CLOUDINARY_API_KEY` | Next.js + Render | Cloudinary API key |
| `CLOUDINARY_API_SECRET` | Next.js + Render | Cloudinary signing secret |

### Screenshot override

```bash
SCREENSHOT_URL_TEMPLATE=https://image.thum.io/get/width/1200/noanimate/{url}
```

## GitHub Actions

Set as repository variables for keep-alive workflows:

| Variable | Purpose |
|----------|---------|
| `VERCEL_APP_URL` | Ping Vercel frontend |
| `RENDER_API_URL` | Ping Render API health |
| `SUPABASE_PROJECT_URL` | Ping the Supabase Auth health endpoint |

## Template file

Copy `env.template` to `.env.local` for local development:

```bash
cp env.template .env.local
```

Replace placeholders locally; do not paste real credentials into documentation, source control, or public build logs.

## Variable resolution

| Context | API base URL |
|---------|-------------|
| `NEXT_PUBLIC_API_URL` empty | Same-origin `/api/*` (Next.js) |
| `NEXT_PUBLIC_API_URL` set | External URL (Render) |

Resolved in `lib/api-base.ts`.
