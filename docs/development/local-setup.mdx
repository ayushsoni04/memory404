Set up a local development environment for memory404.

## Prerequisites

- Node.js 20+
- npm
- A MongoDB Atlas deployment (or compatible local MongoDB replica set)
- Chrome (for extension development)

## Clone and install

```bash
git clone https://github.com/ayushsoni/memory404.git
cd memory404
npm install
```

## Environment

```bash
cp env.template .env.local
```

Set the MongoDB connection:

```bash
MONGODB_URI="mongodb+srv://<user>:<password>@<cluster>/<database>?retryWrites=true&w=majority"
MONGODB_DB="memory404"
```

`MONGODB_DB` is optional and defaults to `memory404`. Keep the real URI out of source control. Add `NEXT_PUBLIC_SUPABASE_URL` and a Supabase publishable key only when testing Auth.

## Database

The app uses the MongoDB native driver. MongoDB creates collections lazily; the Express startup path creates the required indexes for normalized `users`, `groups`, and `links` collections idempotently. Start `npm run backend:dev` once for a new database before relying on the single-stack mode. There is no local SQL migration step.

## Development modes

### Single stack (recommended for most work)

```bash
npm run dev
# http://localhost:3000
```

All API routes served by Next.js at `/api/*`. Includes trash, stats, and image-proxy.

### Split stack (production-like)

```bash
# Terminal 1
npm run dev

# Terminal 2
npm run backend:dev
```

Add to `.env.local`:

```bash
NEXT_PUBLIC_API_URL=http://localhost:4000
```

## Extension development

```bash
cd extension
npm install
npm run dev    # Watch mode
# or
npm run build  # Production build
```

Load `extension/dist` as unpacked in Chrome. Set App URL to `http://localhost:3000`.

## Scripts

| Script | Description |
|--------|-------------|
| `npm run dev` | Start Next.js dev server |
| `npm run backend:dev` | Start Express API on port 4000 |
| `npm run build` | Create a production Next.js build |
| `npm run lint` | Run ESLint |
| `npm run test:metadata` | Test metadata enrichment pipeline |
| `npm run db:mongo:dry-run` | Inspect legacy Postgres source counts |
| `npm run db:mongo:migrate` | Migrate legacy Postgres data to MongoDB |
| `npm run db:mongo:verify` | Verify migrated counts and relationships |

## Project structure

```
memory404/
├── app/                  # Next.js App Router pages and API routes
│   ├── page.tsx          # Main vault (VaultInbox)
│   ├── settings/         # Settings page
│   ├── trash/            # Trash page
│   ├── workspace/        # Screen editor
│   └── api/              # Next.js Route Handlers
├── backend/              # Express API (Render production)
├── components/           # React components
├── extension/            # Chrome MV3 extension
├── lib/                  # Shared utilities
│   ├── db/               # MongoDB client, types, and repositories
│   ├── link-providers/   # Provider-specific metadata rules
│   ├── enrich-link-metadata.ts
│   └── api-base.ts       # API URL resolution
├── scripts/              # Migration and maintenance scripts
└── docs/                 # Mintlify documentation
```

## Linting

```bash
npm run lint
```

ESLint with `eslint-config-next` for Next.js conventions.

## Testing metadata

```bash
npm run test:metadata
```

Runs `scripts/test-metadata.ts` to verify the enrichment pipeline against sample URLs.

## Mintlify docs

Preview documentation locally:

```bash
cd docs
npx mint dev
```

Requires the Mintlify CLI (`npm i -g mint`).
