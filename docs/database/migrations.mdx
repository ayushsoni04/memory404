MongoDB has no application-time SQL migration step. The native driver creates required indexes idempotently through `ensureMongoIndexes()`, while document-shape changes should be handled with explicit, reviewable scripts.

## Commands

| Command | Purpose |
|---------|---------|
| `npm run db:mongo:dry-run` | Read source counts without writing to MongoDB |
| `npm run db:mongo:migrate` | Copy legacy Postgres data and create indexes |
| `npm run db:mongo:verify` | Compare counts and run integrity checks |

## Postgres to MongoDB

<Steps>
  <Step title="Configure source and target">
    Set `DATABASE_URL` only for the legacy source database. Set `MONGODB_URI` for the Atlas target and optionally `MONGODB_DB` (defaults to `memory404`). Never commit either connection string.
  </Step>
  <Step title="Dry run">
    ```bash
    npm run db:mongo:dry-run
    ```
    This validates both connections and prints source counts without writing documents.
  </Step>
  <Step title="Copy during a maintenance window">
    ```bash
    npm run db:mongo:migrate
    ```
    The script upserts normalized `users`, `groups`, and `links` documents while preserving IDs and timestamps, creates indexes, and verifies the result.
  </Step>
  <Step title="Verify independently">
    ```bash
    npm run db:mongo:verify
    ```
    Verification compares collection counts and checks for orphaned users, groups, parent groups, links, and invalid metadata statuses.
  </Step>
</Steps>

## Cutover

After verification:

- Add the same `MONGODB_URI` and `MONGODB_DB` to Vercel and Render.
- Redeploy both services.
- Confirm `/api/health` and `/health`.
- Keep the legacy source read-only until application smoke tests pass.

`DATABASE_URL` is needed only while running the legacy migration script. It is not a runtime variable and should be removed from Vercel and Render after cutover.

<Warning>
The migration uses upserts, but rerunning it after accepting writes in MongoDB can overwrite migrated documents with stale source values. Keep writes paused until migration and verification finish.
</Warning>
